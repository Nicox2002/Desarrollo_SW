Fase Descripcion
Iniciacion
  • Nace la idea del producto de SW
  • Se identifica quien sera nuestros clientes
  • Entrevistar al cliente para realizar las funcionalidades del SW final
  • Analisis para identificar productos similares
  • Determinar si el producto es posible realizarlo y si tendra una buena aceptaciòn
Planeaciòn
  • Se necesita planear los recursos necesarios para implementar el proyecto de SW (Tiempos, Cronogramas, Personal. ETC)
  • Determinar el costo del proyecto y tiempo desarrollo
  • Ajustar el cronograma para la entrega del proyecto
  • Plan de desarrollo para las necesidades del proyecto
  • Plan de Gestion de configuraciòn: (Actividades para controlar los cambios y manejo de versiones)
Analisis
  • Compilar la mayor informacion para las necesidades del cliente
  • Se compila informacion detallada para las funcionalidades del producto, desempeño, confiabilidad, seguridad, usabilidad
  • Requerimientos generales que se pueden compilar por medio de reuniones con los clientes y lluvia de ideas
  • Los requrimientos definen el tipo de producto de SW que debemos hacer
Diseño
  • En esta fase se define como el SW debe ser construido para cumplir con los requerimientos definidos en la fase anterior
  • Definir la arquitectura del SW (Divisiòn en Sub-Sistemas) y se diseña de forma especifica cada componente del SW
  • Crear con ayuda de Sketches o Mockups el diseño de como se vera el producto
Implementaciòn
  • Se programa todo nuestro SW
  • Elobjetivo de esta fase anterior y programarlo completamente usando las tecnologias que se hayan definido
  • Tener en cuenta los componentes para el SW que se necesitan de otros fabricantes
Pruebas
  • Se prueba el SW creado para verificar si el producto cumple con los requrimientos
  • Se divide en 3 Sub-Fases (1. Cada modulo se prueba de forma intdependiente, 2. Los modulos se van integrando para deterinar si la comunicacion entre modulos es correcta. 3. Ya despues de haber probado cada modulo realizar una prueba general de todos los modulos juntos.)
Mantenimiento
  • Ajustes finales en cuenato el SW, para poder liberar oficialmente al Cliente
  • Reparar fallos que se presenten durante el uso por parte de los clientes
  • Se implementan requrimientos adicionales de los clientes para las mejoras
  • Se mejoran atributos del SW como la confiabilidad, desempeño, ETC
  • Determinar el Mantenimiento se genera una nueva version de SW
